## System Administrator Pages


---


### 20 – Admin Home


The **Admin Home** page serves as the central entry point for administrators, providing quick access to all system-level tools.


#### **Key Features**
- Central navigation hub for admin functionality  
- Quick links to core administrative pages  
- Overview of system responsibilities  


#### **Purpose**
Provides a starting point for administrators to navigate and manage the platform.


---


### 21 – Administrator Dashboard


The **Administrator Dashboard** provides a high-level overview of system activity and platform health.


#### **Key Features**
- Registered users, upcoming events, and pending approvals  
- System status indicator  
- System overview (users, events, logs, approvals)  
- Admin controls (refresh interval, maintenance mode, backups, system check)  
- Recent admin activity tracking  


#### **Purpose**
Allows administrators to quickly assess system state and take action when needed.


---


### 22 – System Status


The **System Status** page monitors platform health and activity in real time.


#### **Key Features**
- Live metrics (users, events, logs)  
- System status indicator  
- System overview and activity tracking  


#### **Purpose**
Helps administrators monitor performance and identify issues quickly.


---


### 23 – Backups


The **Backups** page manages system backups and ensures data integrity.


#### **Key Features**
- Run backup action  
- Backup metrics (total, completed, failed)  
- Backup history tracking  


#### **Purpose**
Ensures system data can be reliably stored and recovered.


---


### 24 – Logs


The **Logs** page provides visibility into system activity for auditing and debugging.


#### **Key Features**
- Filter logs by user, action type, or limit  
- Activity log table (user, type, description)  
- Export logs option  


#### **Purpose**
Supports troubleshooting and tracking of system events.


---


### 25 – Admin User Management


The **Admin User Management** page allows administrators to manage users and roles.


#### **Key Features**
- Search users by ID, name, email, role, or status  
- View current users  
- Create, update, and delete users  
- Refresh user data  


#### **Purpose**
Enables control over user access and role management across the platform.


---


### 30 - About Page


The **About** page provides an overview of the CampusConnector platform and its purpose.


#### **Key Features**
- Description of the platform and its goals  
- Explanation of supported user roles:
  - Students  
  - Event Coordinators  
  - Data Analysts  
  - System Administrators  
- Overview of system capabilities and future features  


#### **Purpose**
Provides users with context about the application, its functionality, and the overall system design.

## Student Pages


The following pages are designed for the **Student** role, allowing users to browse events, manage their registrations, and stay updated on campus activities.


---


### 40 – Student Home


The **Student Home** page serves as the main landing page for students, providing quick access to key features.


#### **Key Features**
- Welcome message and overview  
- Quick navigation buttons:
  - Browse upcoming events  
  - View registered events  
  - Check notifications  


#### **Purpose**
Provides a simple starting point for students to explore and interact with the platform.


---


### 41 – Browse Events


The **Browse Events** page allows students to explore upcoming campus events.


#### **Key Features**
- Search events by title  
- Filter events by category  
- View upcoming events list  


#### **Purpose**
Helps students discover events and find ones that match their interests.


---


### 42 – Event Details


The **Event Details** page provides detailed information about a selected event and allows students to register.


#### **Key Features**
- Displays event information:
  - Title  
  - Date and time  
  - Location  
  - Capacity and status  
  - Description  
- Optional event image display  
- **RSVP functionality** for students  
- Navigation back to Browse Events  


#### **Purpose**
Allows students to view full event details and register for events directly.


---


### 43 – My Events


The **My Events** page displays events that a student has registered for.


#### **Key Features**
- View registered events  
- Track upcoming commitments  


#### **Purpose**
Allows students to manage and keep track of events they plan to attend.


---


### 44 – Notifications


The **Notifications** page provides updates and reminders related to student activity.


#### **Key Features**
- View event-related notifications  
- Receive reminders for upcoming events  


#### **Purpose**
Keeps students informed about important updates and event reminders.

##  Event Coordinator Pages


The following pages are designed for the **Event Coordinator** role, allowing users to create events, manage existing ones, and handle attendance.


---


### 50 – Coordinator Home


The **Coordinator Home** page serves as the main landing page for event coordinators, providing quick access to event management tools.


#### **Key Features**
- Welcome message and coordinator overview  
- Quick navigation buttons:
  - Create a new event  
  - Manage existing events  
  - Manage attendance and RSVPs  


#### **Purpose**
Provides a central starting point for coordinators to create and manage campus events.


---


### 51 – Create Event


The **Create Event** page allows coordinators to publish new events through a form-based workflow.


#### **Key Features**
- Enter event details:
  - Event title  
  - Date  
  - Start and end time  
  - Status  
  - Category  
  - Location  
  - Image URL  
  - Description  
- Form validation for required fields  
- Publish event action  


#### **Purpose**
Allows coordinators to create and submit new events to the platform.


---


### 52 – Manage Events


The **Manage Events** page allows coordinators to view, edit, and delete events.


#### **Key Features**
- View list of existing events  
- Display event details (date, time, category, status)  
- Edit events through navigation to the edit page  
- Delete events directly  


#### **Purpose**
Allows coordinators to manage and maintain their events.


---


### 53 – Edit Event


The **Edit Event** page allows coordinators to update information for an existing event.


#### **Key Features**
- Pre-filled form with existing event details  
- Modify:
  - Title  
  - Date and time  
  - Status  
  - Category  
  - Location  
  - Image URL  
  - Description  
- Save changes or return to event list  
- Validation for required fields and time logic  


#### **Purpose**
Ensures event information remains accurate and up to date.


---


### 54 – Manage Attendance


The **Manage Attendance** page allows coordinators to monitor and manage RSVPs for a selected event.


#### **Key Features**
- Select an event to manage  
- View event details (capacity and status)  
- RSVP metrics:
  - Total RSVPs  
  - Approved / registered users  
  - Waitlisted users  
- View list of participants with:
  - Name  
  - Email  
  - RSVP status  
- Update RSVP status (e.g., approve or change status)  
- Remove RSVPs from the event  


#### **Purpose**
Allows coordinators to actively manage event participation and control attendance.
